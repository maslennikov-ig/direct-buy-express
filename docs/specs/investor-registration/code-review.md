# Code Review Report: Phase 7 (Investor Registration)

## Находки и баги (Bugs & Issues)

### 1. Отсутствие валидации `maxBudget >= minBudget` (High Priority)
В спецификации (`docs/specs/investor-registration/spec.md`) было явно указано:
> Шаг 3: Бюджет (max) "Укажите ваш максимальный бюджет (в рублях):"
> Валидация: Только числа (должен быть >= min).

В текущей реализации `bot/conversations/investor-registration.ts` инвестор может ввести `minBudget = 20000000` и `maxBudget = 100`. Код это беспрепятственно пропустит и сохранит в БД, что сломает логику матчинга лотов.

### 2. Ошибка парсинга чисел (Средний приоритет / UX)
Код парсинга бюджета: `parseInt(text.replace(/\D/g, ""), 10)`.
Если пользователь введет "10 млн", `replace` удалит буквы и оставит `10`. В итоге в БД запишется бюджет 10 рублей. Это очень популярный юзкейс для инвесторов. Нужно либо строго требовать ввод полных нулей и выдавать ошибку, либо распознавать "млн". Лучше для начала давать четкую инструкцию или предупреждение, если число слишком маленькое (например, меньше `1 000 000`).

### 3. Отсутствие обработки текстовых сообщений во время ожидания CallbackQuery (Low Priority)
Вызов `await conversation.waitForCallbackQuery(["nda_accept", "nda_reject"])` будет игнорировать любой текст, введенный пользователем, что может привести к зависанию UX (пользователь пишет "Согласен", а бот молчит). Желательно перехватывать и другие сообщения, прося нажать на кнопку.

### 4. Prisma `InvestorProfile` Schema Improvements (Low)
Множественные индексы: Поиск инвесторов будет производиться по `districts`, `minBudget` и `maxBudget`, когда поступает новый лот. Отсутствуют индексы на эти поля в схеме (хотя это можно отложить на фазу оптимизации производительности).

## План действий (Action Items)
