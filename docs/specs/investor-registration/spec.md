# Управление профилем Инвестора (Spec)

## Бизнес-требования
Инвесторы — основная "покупающая" сила на платформе. До начала взаимодействия с реальными объектами, инвестору нужно пройти первичную регистрацию через Telegram-бота.

### 1. Роль и доступы
- После того как пользователь в `/start` выбрал роль `role_investor`, он направляется в диалог регистрации инвестора.
- Статус верификации по умолчанию (после заполнения анкеты) — `isVerified: false`.
- Пока инвестор не верифицирован менеджером, он не получает пуши о новых лотах.

### 2. Шаги диалога (grammY conversations)
- **Шаг 1: NDA.** Пользователю отправляются условия конфиденциальности базы. 
  - Кнопки: `[✅ Принимаю]`, `[❌ Отказываюсь]`.
  - При отказе: диалог прерывается, сообщение "Для продолжения работы в сервисе необходимо принять NDA."
- **Шаг 2: Бюджет (min)** "Укажите ваш минимальный бюджет (в рублях):"
  - Валидация: Только числа.
- **Шаг 3: Бюджет (max)** "Укажите ваш максимальный бюджет (в рублях):"
  - Валидация: Только числа (должен быть >= min).
- **Шаг 4: Районы** "Укажите интересующие районы (через запятую):"
  - Сохраняется как массив строк.
- **Шаг 5: Завершение.**
  - Данные сохраняются в БД.
  - Сообщение: "Ваша анкета на модерации. Мы сообщим, когда администратор одобрит ваш профиль."

### 3. Данные (Модель БД)
`InvestorProfile`:
- `id`: PK
- `userId`: FK (`User`)
- `minBudget`: Decimal
- `maxBudget`: Decimal
- `districts`: String[]
- `isVerified`: Boolean (default: false)
