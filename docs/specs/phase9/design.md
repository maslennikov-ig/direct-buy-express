# Phase 9: Document Pipeline & Manager Admin Panel — Design

## 1. Контекст

После завершения аукциона (Phase 8) собственник выбирает победителя. Далее нужно:
1. Собственник загружает документы через бот.
2. Менеджер проверяет документы в админ-панели.
3. При одобрении/отклонении → менеджер связывается с участниками вручную.

**Бизнес-решение:** Автоматический депозит (50к ₽) откладываем. Переходим сразу к загрузке документов.

## 3. Document Pipeline (Телеграм-бот)

### 3.1 Триггер
- Собственник выбрал победителя аукциона → `Lot.status` = `WAITING_DOCS`, `Lot.winnerId` = ID инвестора.
- Бот отправляет собственнику сообщение: «Загрузите документы в течение 2 часов».
- Запускается 2-часовой SLA-таймер (`BullMQ job: SLA_DOCS_UPLOAD`).

### 3.2 Conversation Flow (`uploadDocsConversation`)
1. **ЕГРН:** «Отправьте выписку из ЕГРН (файл PDF или фото).» → ожидание `document` или `photo`.
2. **Паспорт:** «Отправьте фото главного разворота паспорта собственника.» → ожидание `photo`.
3. **Документ основания:** «Отправьте документ основания (ДКП, приватизация и т.п.).» → ожидание `document` или `photo`.
4. **Отказники (опционально):** «Если была приватизация — отправьте отказ от приватизации. Если нет — напишите "Нет".» → ожидание `document`/`photo`/текст «нет».

### 3.3 Storage
- Файлы скачиваются через Telegram API (`ctx.api.getFile` → `downloadFile`).
- Сохраняются в `./uploads/lots/{lotId}/`.
- Создаются записи `Media` в БД с типами: `EGRN`, `PASSPORT`, `OWNERSHIP_DOC`, `PRIVATIZATION_REFUSAL`.

### 3.4 Финализация
- `Lot.status` → `DOCS_AUDIT`.
- SLA-таймер `SLA_DOCS_UPLOAD` снимается.
- Собственнику: «✅ Документы загружены. Менеджер проверит их в ближайшее время.»
- Инвестору-победителю: «Собственник загрузил документы. Ожидайте результат аудита менеджером.»

### 3.5 SLA-таймер
- `SLA_DOCS_UPLOAD` — 2 часа.
- При просрочке: уведомление менеджеру в отдельный чат.

## 4. Manager Admin Panel (Next.js)

### 4.1 Авторизация
- Middleware: проверка cookie `admin_session` (простой токен из `.env`).
- Страница логина: `/admin/login` — форма ввода пароля.
- При верном пароле — устанавливаем cookie, редирект на `/admin`.

### 4.2 Модерация инвесторов (`/admin/investors`)
- **Текущая страница уже есть** — mock-данные.
- **Доработка:** Подключить к Prisma (`InvestorProfile` где `isVerified = false`, join `User`).
- **API:** `POST /api/admin/investors/[id]/verify` — ставит `isVerified = true`.
- Кнопка «Одобрить» вызывает API и обновляет UI.

### 4.3 Аудит документов (`/admin/lots`)
- **Текущая страница уже есть** — mock-данные.
- **Доработка:** Фильтр по статусу `DOCS_AUDIT`. Подключить к Prisma (include `Media`, `owner`, winning `Bid`).
- **Детали лота:** Показывать загруженные документы (изображения).
- **Действия:**
  - «Одобрить» → `POST /api/admin/lots/[id]/approve` → `status = READY_TO_DEAL`. Бот шлёт: «Менеджер поможет назначить встречу.»
  - «Отклонить» → `POST /api/admin/lots/[id]/reject` → `status = WAITING_DOCS`. Бот шлёт: «С вами свяжется менеджер.»

### 4.4 Защищённая отдача файлов
- API роут: `GET /api/media/[id]` — проверяет авторизацию (менеджер), отдаёт файл из `./uploads/`.
