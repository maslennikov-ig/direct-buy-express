**Контекст проекта:**
Мы разрабатываем **Direct Buy Express** — P2P-платформу в Telegram для быстрого выкупа недвижимости инвесторами напрямую у собственников без посредников.
**Стек:** TypeScript, Node.js, Next.js 15 (App Router), React 19, TailwindCSS 4, shadcn/ui, PostgreSQL + Prisma, grammY (Telegram Bot), BullMQ + ioredis для SLA-таймеров.
Проект использует строгий **TDD** (Test-Driven Development) с `vitest` на всех этапах и разделяет процессы через `Subagent-Driven Development`. У нас работает issue-трекер `bd` (Beads) для ведения задач.

**Текущий статус (Завершено: Фазы 1-8):**
Реализован фундамент бота, аукционный движок (Auction Engine), фоновые таймеры (BullMQ), мэтчинг инвесторов и их предварительная регистрация. Аукцион успешно собирает ставки и определяет победителя среди тысяч инвесторов.

**Твоя задача — реализовать Phase 9: Document Pipeline & Admin Panel.**
По решению бизнеса мы *временно отказываемся от механики автоматического гарантийного взноса (депозитов)*, поэтому переходим сразу к загрузке документов и модерации.

**Требования к Phase 9:**
1. 📦 **Document Pipeline (Телеграм-бот):**
   - Реализовать `uploadDocsConversation` в боте (`bot/conversations/upload-docs.ts`).
   - По правилам SLA, после выбора победителя аукциона Собственник должен прислать фото своего паспорта и выписку из ЕГРН в течение 2 часов.
   - Бот должен корректно принимать медиафайлы (фото/файлы), сохранять их локально (пока в папку `uploads` проекта или аналогично) и создавать записи в таблице `Media` БД Prisma, привязывая их к лоту.
   - По завершении загрузки статус лота меняется на `DOCS_AUDIT`. Бот уведомляет выигравшего Инвестора о начале аудита документов менеджером.
2. 🛡 **Manager Admin Panel (Web Next.js):**
   - Реализовать защищенный дашборд администратора (Менеджера) на Next.js в папке `app/admin`. Желательно с базовой авторизацией (хотя бы статичным паролем / HTTP Basic, или простой сессией, если не указано иное).
   - **Раздел "Модерация инвесторов":** Выводить список пользователей с `role = INVESTOR` и `isVerified = false`. Сделать кнопку/API-роут для перевода их в статус `true`. Без этого они не получают рассылки новых лотов из аукциона.
   - **Раздел "Аудит лотов":** Выводить список лотов в статусе `DOCS_AUDIT`, где менеджер может просмотреть привязанные медиафайлы (паспорт, ЕГРН).
   - Интерфейс админки должен быть красивым и функциональным (используй `shadcn/ui` и `TailwindCSS 4`).
