# Phase 8: Auction Engine & Investor Bidding (Design)

## 1. Цель
Реализовать ядро P2P платформы (Аукцион), позволяющее инвесторам получать уведомления о новых лотах, делать ставки (офферы) и автоматически закрывать аукцион при достижении лимитов (12 часов или 5 ставок).

## 2. Архитектура и Механика (Data Flow)

**A. Публикация лота (Broadcaster & @grammyjs/auto-retry):**
- Как только лот переводится в статус `AUCTION` (Менеджером или автоматически), вызывается функция `broadcastLotToInvestors(lotId)`.
- Она ищет всех инвесторов (`InvestorProfile`), у которых `isVerified == true` и их `maxBudget` / `districts` пересекаются с параметрами лота.
- **Инструмент:** Для массовой рассылки будет использован официальный плагин `@grammyjs/auto-retry`. Он автоматически ставит запросы на паузу, если Telegram возвращает ошибку `429 Too Many Requests`, обеспечивая 100% доставку без написания сложных очередей и ручной обработки лимитов (limits: 30 msg/sec).
- Бот в цикле рассылает красивую карточку лота с Inline-кнопкой "💰 Предложить цену".

**B. Процесс ставки (Bidding Conversation):**
- Инвестор нажимает "Предложить цену".
- Запускается grammY `conversation` (`makeBidConversation`).
- Инвестор вводит сумму (Total). Скрытых комиссий в UI инвестора нет.
- Схема: `prisma.bid.create({ lotId, investorId, amount })`. Инвестор получает подтверждение.
- Сразу после создания ставки проверяется условие: `if (bids.length >= 5) { closeAuction(); }`.

**C. Закрытие Аукциона (Auction Engine Closing):**
- Аукцион закрывается либо по SLA таймеру BullMQ (12 часов) в `worker.ts`, либо досрочно (5 ставок).
- При закрытии: статус лота меняется на `WAITING_CHOICE`, всем инвесторам кто сделал ставку приходит "Сбор предложений завершен", а Собственнику отправляется список из 5 предложений для выбора.

## 3. Задачи (Beads) для делегирования

Мы разделим работу на независимые компоненты для параллельных Субагентов:

1. **Bead 1 (Backend/Queries):** Реализовать функцию матчинга инвесторов и рассылки (Broadcaster). Поиск верифицированных инвесторов в Prisma с учетом бюджета.
2. **Bead 2 (Bot/Conversations):** Реализовать UI-диалог `makeBidConversation` (кнопка, парсинг суммы ставки, запись `Bid` в БД, валидация 5 ставок).
3. **Bead 3 (SLA Worker):** Обновить BullMQ worker `CLOSE_AUCTION`. Добавить отмену таймера при получении 5 ставок, и рассылку итоговых ставок Собственнику.

## 4. Бизнес-правила (Утверждено)
- **Матчинг районов:** Если у инвестора не указаны районы (пустой массив), он получает все лоты по бюджету. Район "Любой" означает получение всех лотов, проходящих по бюджету.
- **Изменение ставки:** Инвестор может изменить свою ставку (Edit Bid) сколько угодно раз (или удалить), пока аукцион не закрыт либо по таймеру (12ч), либо по лимиту (5 ставок суммарно со всех инвесторов).
