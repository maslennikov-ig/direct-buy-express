# 🧪 Гайд по тестированию — Direct Buy

## Предварительные требования

**Перед началом тестирования необходимо:**
1. Получить **BOT_TOKEN** от @BotFather и прописать его на сервере
2. Узнать **MANAGER_CHAT_ID** (ID чата менеджера)
3. Настроить DNS для `directbuy.symancy.ru` (или тестировать по IP)

| Компонент | URL |
|---|---|
| Админ-панель | `http://91.132.59.194:3001/admin` |
| Логин | `http://91.132.59.194:3001/admin/login` |
| Telegram-бот | Ссылка на бота (по BOT_TOKEN) |

**Пароль админки:** `DirectBuyAdmin2026`

## Часть 1: Telegram-бот

### 🟢 TC-1: Старт и выбор роли

**Шаги:**
1. Откройте бота в Telegram
2. Нажмите `/start`

**Ожидается:**
- Приветственное сообщение с описанием платформы
- Кнопка **«✅ Принимаю условия и начинаю»**

3. Нажмите кнопку принятия условий

**Ожидается:**
- Сообщение «Кто вы? Выберите роль:»
- 3 кнопки: **👤 Я Собственник**, **🤝 Я Брокер**, **💰 Я Инвестор**

### 🟢 TC-2: Создание лота (Собственник/Брокер)

> Используйте 2-й аккаунт Telegram или основной

**Шаги:**
1. `/start` → Принять условия → Нажать **«👤 Я Собственник»**
2. Бот просит адрес → Ввести: `Москва, Тверская 15`

**Ожидается:** Бот предложит стандартизированный адрес через DaData (если ключ настроен) или примет как есть.

3. Ввести площадь: `65.5`
4. Ввести этаж: `7`
5. Ввести кол-во комнат: `2`
6. Есть ли долги? → `Нет`
7. Есть ли ипотека? → `Нет`
8. Есть ли зарегистрированные? → `Нет`
9. Причина срочности → `Переезд`
10. Ожидаемая сумма → `8000000`

**Ожидается:** Бот покажет резюме карточки:
```
📊 Ваш лот:
📍 Адрес: Москва, Тверская 15
📏 65.5 м², Этаж 7, 2 комн.
💰 Ожидаемая сумма: 8 000 000 ₽
```

11. Отправить **7 фотографий** объекта (по одной)

**Ожидается:** После 7 фото появляется кнопка **«📤 Готово — загрузить фото»**

12. Нажать «Готово» (или отправить до 10 фото, затем автозавершение)

**Ожидается:**
- Лот создан, бот уведомляет: «✅ Лот создан и выставлен на аукцион!»
- В БД: `Lot.status = "AUCTION"`, `auctionEndsAt` = через 12 часов
- Таймер `CLOSE_AUCTION` поставлен в BullMQ

**Негативные сценарии:**
- Ввести площадь `абвгд` → Бот переспросит «введите корректное число»
- Ввести 0 комнат → Принимается (студия)
- Отправить текст вместо фото → Бот попросит фото

### 🟢 TC-3: Регистрация инвестора

**Шаги:**

2. Принять NDA → Кнопка **«✅ Принимаю»**

**Негативные сценарии:**

### 🟢 TC-4: Аукцион и ставки

> Требуется: лот в статусе AUCTION + верифицированный инвестор

**Предусловие:** Верифицируйте инвестора в админке (TC-11)

**Шаги:**
1. Бот автоматически рассылает уведомление подходящим инвесторам:
   ```
   📢 Новый объект! Москва, Тверская 15, 65.5 м², 2 комн.
   [Сделать ставку]
   ```
2. Инвестор нажимает **«Сделать ставку»** (`bid_lot_<id>`)
3. Вводит сумму ставки: `7500000`

**Ожидается:**
- Ставка создана (`Bid` в БД)
- Собственник получает уведомление о новой ставке

**Ограничение:** Максимум 5 ставок на лот. После 5-й — аукцион закрывается автоматически.

---

### 🟢 TC-5: Закрытие аукциона (SLA — 12 часов)

**Вариант A — Автоматическое (таймер):**
- Через 12 часов Worker выполняет `CLOSE_AUCTION`
- Статус лота → `WAITING_CHOICE`
- Собственник получает список ставок с кнопками выбора

**Вариант B — При 5 ставках:**
- Как только 5 инвесторов сделали ставки → аукцион закрывается
- Тот же результат

**Шаги:**

### 🟢 TC-7: Загрузка документов собственником

**Шаги:**
1. Собственник нажимает кнопку **загрузки** → Входит в диалог
2. Отправляет 4 документа (фото/PDF):
   - 📄 Выписка ЕГРН
   - 📄 Паспорт
   - 📄 Документ основания (ДКП/приватизация)
   - 📄 Отказ от приватизации (или написать «Нет»)

**Ожидается:**
- 4 записи `Media` в БД с типами: `EGRN`, `PASSPORT`, `OWNERSHIP_DOC`, `PRIVATIZATION_REFUSAL`

- Собственник: «✅ Документы загружены!»

**Негативные сценарии:**

### 🟢 TC-8: Решение инвестора по документам

**Предусловие:** Менеджер одобрил документы → Лот в `INVESTOR_REVIEW`

**Шаги:**
1. Инвестор видит кнопки:
   ```
   [✅ Одобряю] [❌ Отклоняю]
   ```
2. Нажимает **«✅ Одобряю»**

**Ожидается:**
- Кнопки исчезают, текст: «✅ Вы одобрили документы. Ожидайте связи от менеджера.»
- Лот → `MANAGER_HANDOFF`
- Собственник получает уведомление
- Менеджер получает уведомление в `MANAGER_CHAT_ID`

**При отклонении:** Тот же переход (MANAGER_HANDOFF), менеджер разруливает ситуацию.

## Часть 3: SLA-таймеры (BullMQ)

| Таймер | Задержка | Что происходит |
|---|---|---|
| `CLOSE_AUCTION` | 12 ч | Лот `AUCTION → WAITING_CHOICE` |
| `SLA_OFFER_RESPONSE` | 2 ч | Алерт менеджеру, если собственник не выбрал |
| `SLA_DOCS_UPLOAD` | 2 ч | Алерт менеджеру, если собственник не загрузил доки |
| `SLA_INVESTOR_REVIEW` | 2 ч | Алерт менеджеру, если инвестор не ответил |

**Как тестировать:** Эти таймеры работают с реальными задержками. Для теста можно:
- Проверить, что таймеры создаются (логи worker'а)
- Дождаться истечения таймера (или уменьшить delay в коде)

## Известные ограничения

1. **BOT_TOKEN** — бот не запущен до ввода реального токена
2. **DaData** — без ключа `DADATA_API_KEY` подсказки адресов не работают (не критично)
3. **Файлы документов** — в тестовом режиме создаются пустые файлы-заглушки
4. **SLA-таймеры** — в production работают с реальными задержками (2-12 часов)
